## arr.sort() vs sorted(arr)

### arr.sort()

- 원본을 직접 정렬(원본 변경)
- 반환값 없음
- 메모리 효율 좋음

### sorted(arr)

 - 새로운 정렬된 리스트 생성 및 반환 (원본 유지)
 - 함수형 스타일
 - 원본 비교할 때, 여러 기준 정렬할 때 사용


 ### 내림차순

 ```python
    arr.sort(reverse=True)
    sorted(arr, reverse=True)
 ```
### Key 정렬(람다)

```python
    arr = [(1,3), (2,1), (3,2)]
    arr.sort(key = lambda x: x[1])
```
### lambda x: x[1]의 의미

각 원소에서 두 번째 값만 꺼내라.

#### 결과 :

두 번째 값 기준으로 정렬됨

```
[(2,1), (3,2), (1,3)]
```

### 두 개 기준 정렬

```python
    arr.sort(key = lambda x: (x[1],x[0]))
```

1. x[1] 먼저
2. 같으면 x[0]


### 문자열 정렬 중 길이 정렬

```python
arr.sort(key=len)
```

### 대소문자 무시 정렬

```python
arr = ["a", "B", "c"]
arr.sort(key=str.lower)
```


### 절댓값 기준 정렬

```python
arr.sort(key=abs)
```

### 복합 정렬


```python
arr.sort(key=lambda x: (x[0], -x[1]))
```

x[0] : 오름차순
-x[1] : 내림차순

의미: 첫 번째 오름차순, 두 번째 내림차순

### 리스트 말고도 정렬 가능

```python
sorted("cba")  # ['a','b','c']
sorted({3,1,2})  # [1,2,3]
```

### 시간복잡도

sort() / sorted() -> O(n log n)

내부 알고리즘: Timesort
