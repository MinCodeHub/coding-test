## 📌 문제 해결 회고 – 큰 수 만들기

### 1️⃣ 초기 접근

처음에는 `prev` 변수를 사용하여 **이전 값과 현재 값을 비교하는 방식**으로 문제를 해결하려고 했다.
현재 값이 이전 값보다 크면 리스트에 추가하고, 작으면 제거 횟수(`cnt`)를 증가시키는 방식으로 구현했다.

하지만 이 방식은 **이전 값 하나만 기준으로 판단**하기 때문에,
여러 개의 숫자를 연속적으로 제거해야 하는 상황을 처리하지 못했다.

---

### 2️⃣ 문제 발생 원인

구현 과정에서 다음과 같은 문제가 발생했다.

* **이전 값 하나만 비교하는 구조**

  * 실제 문제는 이전 값 하나가 아니라, **이전의 여러 값들과 비교**해야 함
* **한 번에 하나만 제거 가능**

  * `if` 조건을 사용하여 제거 횟수를 1씩 증가시키는 구조 → 연속 제거 불가능
* **정답이 생성되지 않는 경우 발생**

  * `cnt == k` 조건을 만족하지 못하면 `answer`가 빈 문자열로 반환됨

특히, `"4177252841"`과 같은 입력에서
여러 개의 작은 숫자를 한 번에 제거해야 하는 상황을 처리하지 못해 오답이 발생했다.

---

### 3️⃣ 깨달은 점

이 문제는 단순 비교 문제가 아니라,

> **“앞자리에서 작은 숫자를 제거하여 가장 큰 수를 만드는 그리디 문제”**

라는 것을 깨달았다.

또한, 이전 값 하나가 아니라
**이전 값들의 집합을 관리해야 한다는 점에서 스택 자료구조가 필요함**을 이해했다.

---

### 4️⃣ 해결 방법

스택을 사용하여 현재 숫자보다 작은 이전 숫자들을 반복적으로 제거하도록 수정했다.

* 현재 숫자(`num`)가 들어올 때
* 스택의 마지막 값(`stack[-1]`)보다 크면
* `while`문을 통해 조건을 만족하는 동안 계속 제거

또한, 반복문 이후에도 제거 횟수가 남아있을 경우
리스트 뒤에서 남은 개수를 제거하도록 처리했다.

---

### 5️⃣ 최종 코드

```python
def solution(number, k):
    stack = []
    
    for num in number:
        while k > 0 and stack and stack[-1] < num:
            stack.pop()
            k -= 1
        stack.append(num)
    
    if k > 0:
        stack = stack[:-k]
    
    return ''.join(stack)
```

---

### 6️⃣ 핵심 학습 포인트

* “이전 값 비교”가 아니라 **“이전 값들 전체 관리”가 필요한 문제**
* **연속적인 제거가 필요한 경우 → `while` 사용**
* **순서 유지 + 일부 제거 + 최댓값 → 스택 + 그리디 패턴**
* 반복문 이후 남은 처리까지 고려해야 완전한 해결 가능

> 스택을 활용한 그리디 방식으로 문제를 해결할 수 있었다.
